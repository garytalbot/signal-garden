# Signal Garden launch kit

Signal Garden is a tiny browser gallery where every UTC day gets a shared glowing field, the latest broadcasts stay visible as a lightweight archive, and every click can still plant a new procedural bloom with its own shape, palette, and ridiculous little name.

## Show HN version

### Suggested title

Show HN: Signal Garden — a tiny browser gallery with a shared daily signal

### Submission text

I made a small browser toy / gallery called Signal Garden.

Every UTC day gets one shared deterministic field, and the latest broadcasts stay visible on the page as a lightweight archive. You can also ignore all that and just click around to plant your own glowing procedural blooms with generated names.

A few things I liked while building it:
- the daily signal gives the project one common field per day instead of only one-off personal scenes
- the archive makes the project feel more like a living object than a novelty button
- custom gardens can still be replayed, shared by URL, or exported as a PNG

It’s just HTML, CSS, and vanilla JS — no framework, no account, no setup.

Live demo: https://garytalbot.github.io/signal-garden/
GitHub: https://github.com/garytalbot/signal-garden

Would love feedback on both the interaction and where you’d take it next. I’m considering deeper archive browsing, seasonal palette modes, ambient sound, and richer exports.

## Indie Hackers version

### Suggested title

I shipped Signal Garden, a tiny browser gallery with a shared daily signal and replayable procedural blooms

### Post body

Shipped a weird little thing this morning: Signal Garden.

It started as a browser-native art toy where you click into a dark field and plant glowing procedural blooms. The project got more interesting once it had a public rhythm, so now every UTC day gets a shared field and the newest broadcasts stay visible on the page as a lightweight archive.

Live here:
https://garytalbot.github.io/signal-garden/

Code here:
https://github.com/garytalbot/signal-garden

What’s in the current version:
- click-to-plant glowing blooms
- daily signal mode with one shared broadcast garden per UTC day
- a lightweight archive/gallery of the latest broadcasts
- replayable and shareable custom gardens
- one-click PNG export
- plain HTML, CSS, and JS

What I learned:
- tiny projects get easier to talk about once they have a recurring public hook
- a shared daily state is a much better social wedge than only private permalinks
- the archive makes the repo feel active even for brand-new visitors

I’d especially love feedback on distribution and whether this reads more like a pure art toy, a relaxing casual experience, or the early version of something larger.

## Media notes

- For social feeds, attach `assets/launch/community-poster.png` if you want something clean and branded, or `assets/launch/demo-ui.png` if you want the real interface on screen.
- For image replies and follow-ups, a fresh in-app PNG export still works best because it proves the garden is real and interactive.
- For communities that prefer “just the link,” use the main live site in the post and drop a daily signal or custom-garden permalink in a follow-up comment.
- If somebody asks for proof it is not a static mockup, reply with a fresh PNG export or a custom garden permalink.
- If somebody asks what changed recently, lead with the main app URL so the archive sells the story immediately.

## Tone guardrails

- Keep it playful and concrete.
- Call it a browser toy, gallery, art toy, or interactive piece — not a platform.
- Don’t oversell it like it’s venture-backed lawn software.
- The honest angle is strong: it is a strange, lightweight, web-native thing that people can immediately play with.
